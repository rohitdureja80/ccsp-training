# Cloud Migration Strategies

**Domain:** 1 - Cloud Concepts, Architecture & Design  
**Priority:** 🟡 Important  
**Exam Frequency:** ⭐⭐⭐

## Overview

Cloud migration strategies describe how applications and workloads move from one environment to another.

## Common Migration Strategies

| Strategy | Description | Memory Aid |
|---|---|---|
| Rehost | Move workload with minimal change. | Lift and shift |
| Replatform | Make small cloud optimizations without major redesign. | Lift, tinker, and shift |
| Refactor | Redesign application for cloud-native capabilities. | Re-architect |
| Repurchase | Replace with SaaS or commercial product. | Buy instead of build |
| Retire | Decommission no-longer-needed systems. | Turn it off |
| Retain | Keep system as-is for now. | Leave it where it is |

## CCSP Exam Tips

- Rehost = fastest migration, least cloud optimization.
- Refactor = most effort, most cloud-native benefit.
- Repurchase = move to SaaS.
- Retire = remove unnecessary applications.
