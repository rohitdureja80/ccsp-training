# Domain 1 Exam Traps

**Domain:** 1 - Cloud Concepts, Architecture & Design  
**Priority:** 🔴 Critical  
**Exam Frequency:** ⭐⭐⭐⭐⭐

## Trap 1: Data Responsibility

Question asks who is ultimately responsible for protecting customer data.

Answer: **Cloud Customer**

Even in SaaS, the customer remains accountable for data ownership, classification, legal obligations, and appropriate use.

## Trap 2: IaaS vs PaaS

If the customer manages the operating system, think **IaaS**.

If the CSP manages the operating system and runtime, think **PaaS**.

If the customer only uses the application, think **SaaS**.

## Trap 3: Portability vs Interoperability

| Term | Meaning |
|---|---|
| Portability | Ability to move data or workloads between environments. |
| Interoperability | Ability of systems to work together. |

## Trap 4: Broker vs Carrier vs Auditor

| Role | Exam Clue |
|---|---|
| Broker | Helps select, aggregate, or manage services. |
| Carrier | Provides network connectivity. |
| Auditor | Provides independent assessment. |

## Trap 5: Governance vs Management

Governance asks whether the organization is doing the right things.

Management asks whether things are being done correctly.

## Trap 6: Cloud Is Not Automatically More Secure

Cloud can improve security through automation, central IAM, logging, and managed services, but poor configuration can create major risk.

## Trap 7: Shared Responsibility Changes by Service Model

As you move from IaaS to PaaS to SaaS, the CSP manages more of the stack, but the customer never gives up responsibility for its data and governance obligations.
