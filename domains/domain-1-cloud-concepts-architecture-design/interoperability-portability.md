# Interoperability and Portability

Priority: 🟡 Important  
Exam Frequency: ⭐⭐⭐⭐

## Definitions

## Interoperability

Interoperability is the ability of systems, applications, or services to work together across environments.

Example:

A workload in Azure communicates with a database in AWS using standard APIs.

## Portability

Portability is the ability to move applications, data, or workloads from one environment to another with minimal change.

Example:

A containerized application can move from Google Kubernetes Engine to Azure Kubernetes Service.

## Key Difference

| Concept | Main Question |
|---|---|
| Interoperability | Can systems work together? |
| Portability | Can I move it elsewhere? |

## Why This Matters

These concepts reduce vendor lock-in and support multi-cloud or hybrid cloud strategies.

## Exam Tip

If a question emphasizes systems communicating across platforms, think interoperability. If it emphasizes moving workloads or data, think portability.
